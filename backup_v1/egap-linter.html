Installation

Up and running in three steps

Requires Python 3.10 or later. Works on macOS, Linux, and Windows. Runs fully offline — no data leaves your machine.

01

Install the package

From PyPI using pip:

pip install replication-linter

Or install the dev version from source:

git clone https://github.com/bracluque/replication-linter pip install -e .
02

Point it at your project

Run from anywhere — pass the path to your replication package folder:

repro-lint /path/to/your/project

Or from inside the folder:

cd my_paper/ repro-lint .
03

Read the report

Open the HTML report in any browser — no internet required:

open my_paper/00_admin/replication_audit.html

The report maps every finding to a specific journal requirement and fix.

Pre-submission one-liner: Run repro-lint . --strict from inside your replication package. Exit code 0 = clean. Exit code 1 = fix before submitting. Add this to your CI pipeline or run it manually before each submission round.

Common invocations

# Basic audit repro-lint ./my_paper # Strict mode — warnings count as failures (good for CI) repro-lint ./my_paper --strict # Skip PII scan (much faster, use for quick structure checks) repro-lint ./my_paper --no-pii # Suppress codes you've consciously accepted repro-lint ./my_paper --suppress F008 --suppress C014 # Output to a custom directory repro-lint ./my_paper --output-dir /tmp/audit_results # JSON report for programmatic use repro-lint ./my_paper --json # Check only folder structure (fastest) repro-lint ./my_paper --no-pii --no-code
Check modules

Three layers of reproducibility checks

Every finding has a stable code, a severity level, and a one-line fix instruction mapped to a specific journal requirement. Nothing is a vague suggestion.

Module 1 · Folder

Folder structure checker

Audits your project directory against the DCAS standard and SSDE Template README v1.1. Detects missing README files, absent standard directories, duplicate files, and orphaned files.

F — Folder checks

F001error
No README file found at project root
Add a README.md — required by AEA, AJPS, APSR, QJE
F002error
README is empty or whitespace only
Add content — at minimum: data sources, software, run order
F003warn
README is not a PDF or Markdown file
Convert to README.pdf or README.md (AEA requires PDF)
F004error
SSDE Template README section missing
Add the missing heading — required by DCAS v1.0
F005warn
No master script found
Add master.do or run_all.R (required when >4 steps)
F006warn
Expected standard directory missing
Create 00_admin/, 01_data/, 02_code/, 03_output/
F007info
Duplicate file detected (same SHA-256)
Remove the duplicate to reduce confusion
F008info
Orphan file at project root
Move into 01_data/, 02_code/, or 03_output/
F009warn
No data dictionary / codebook found
Add codebook.md — required by APSR and AJPS
F011warn
README has fewer than 3 of 7 SSDE required sections
Expand README using the SSDE Template README v1.1
F012info
No LICENSE file found
Add LICENSE — MIT or CC-BY 4.0 recommended by AEA
Module 2 · PII

Data management & PII safety

Opens CSV, Stata .dta, and Excel files and flags columns or values that suggest personally identifiable information. Uses Stata variable labels, not just column names. Conservative — only flags likely risks.

P — PII checks

P001error
Column name exactly matches PII identifier
Verify column doesn't contain real IDs; pseudonymise or drop
P002warn
Column name partially matches PII pattern
Review column — likely contains sensitive identifiers
P003warn
Column values match PII pattern (≥5% of sampled rows)
Review values for email, phone, national ID patterns
P004warn
Free-text column detected (mean length >40 chars)
Review for identifiable text; consider redaction
P005warn
No codebook found for this data file
Add a codebook describing all variables in this file
P006info
File could not be read (format or encoding error)
Verify file is a valid CSV/DTA/XLSX and not password-protected
P007info
.dta file without a CSV equivalent
Export a CSV alongside the .dta (JPE requires non-proprietary formats)
P008error
Column matches national ID pattern (DNI, cedula, CPF, RUT, CURP, SSN)
Remove or replace with an anonymous key before deposit
Supports Spanish & English. PII patterns cover Latin American IDs (DNI/Peru, cédula/Colombia, CPF/Brazil, RUT/Chile, CURP/Mexico) alongside US and European identifiers.
Module 3 · Code

Code checker — R & Stata

Static analysis of .R, .Rmd, .do, and .ado files. Detects absolute paths, missing seeds, unpinned packages, absent version declarations, dead file references, and more. Never executes your code.

C — R checks

C001error
Absolute path in R file
Replace with here::here() or a relative path
C002warn
Random call without set.seed() in same file
Add set.seed(<integer>) before first random call
C003warn
library() call for package not in renv.lock
Run renv::snapshot() to pin the package
C004warn
No renv.lock file found in project
Run renv::init() then renv::snapshot()
C005info
Output file written by script not present in project
Run the script to generate the file, then include it
C006warn
source() references a non-existent file
Verify the file exists relative to the script or project root
C007info
setwd() call detected
Replace with here::here() for cross-machine compatibility

C — Stata checks

C010error
Hard-coded cd with absolute path
global root "path" then cd "$root"
C011warn
No version statement in .do file
Add version 17.0 as the first command
C012warn
Random command without set seed in same file
Add set seed <integer> before first random command
C013error
Absolute path literal in Stata code
Use "$root/subdir/file.dta" with a root global
C014info
No log using in long do-file (>50 lines)
Add log using "logs/run.log", replace for audit trail
C015warn
do "file" references a non-existent .do file
Verify the file exists relative to $root

C — Cross-file checks

C020warn
No master script with >3 code files
Add master.do or run_all.R calling all scripts in order
C021info
Output name closely matches input name in another script
Verify these are the same file with consistent naming

Suppressing individual findings

Add an inline comment to silence a specific finding on a single line. Suppressed findings are computed but not emitted and don't affect pass/fail.

# R — suppress C001 on this line setwd("/home/ronny/special_path") # repro-lint: ignore C001 /* Stata — suppress C010 on this line */ cd "/Users/brayan/project" /* repro-lint: ignore C010 */

Configuration

Customise with .linterrc

Create a .linterrc file in TOML format at your project root to override defaults. Config is merged with CLI flags — flags take precedence.

Example .linterrc

# .linterrc — TOML format [linter] pii_sample_rows = 300 max_file_size_mb = 100.0 suppressed_codes = ["F008", "C014"] ignore_patterns = ["*.log", "*.tmp"] [linter.folder] expected_top_dirs = [ "00_admin", "01_data", "02_code", "03_output", "04_docs" ] [linter.pii] pii_deep_scan = false [linter.code] check_seed = true check_renv = true check_dead_refs = true

Recommended folder structure

my_paper/ ├── README.pdf # all 7 SSDE sections ├── LICENSE ├── .linterrc ├── 00_admin/ # admin & reports ├── 01_data/ │ ├── raw/ # original data, untouched │ ├── clean/ # processed analysis data │ └── codebook.md # variable descriptions ├── 02_code/ │ ├── master.do # or run_all.R │ ├── 01_clean.do │ ├── 02_analysis.do │ └── renv.lock # for R projects └── 03_output/ ├── table1.tex └── fig1.pdf

CLI flags reference

--output-dir PATHWhere to write the HTML report and manifest. Default: project/00_admin/
--no-folderSkip the folder structure agent (Module 1)
--no-piiSkip PII scanning (Module 2). Faster; use for quick code checks
--no-codeSkip code analysis (Module 3)
--suppress CODESuppress a finding code globally. Repeatable. e.g. --suppress F008
--strictExit code 1 on any WARNING (not just ERROR). Recommended for CI
--quietPrint only errors and the final summary line
--jsonWrite a JSON report instead of HTML
--versionShow version and exit

Exit codes

0 — PassedNo errors found (warnings present unless --strict)
1 — FailedErrors found, or warnings with --strict
2 — Config errorBad path, unreadable config, or conflicting flags
3 — Internal errorUnrecoverable crash. Please open a GitHub issue

Use in GitHub Actions

# .github/workflows/repro.yml name: Replication check on: [push, pull_request] jobs: lint: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - run: pip install replication-linter - run: repro-lint . --strict --no-pii
Project

About repro-lint

repro-lint is an open-source tool developed under an EGAP Innovation Grant in the Open Science and Reproducibility category. It is built by researchers who routinely submit to AEA journals, AJPS, and APSR, and who have experienced first-hand the friction of first-round rejection for avoidable technical issues.

The tool operationalises the DCAS v1.0 standard and the SSDE Template README v1.1 into machine-readable checks. Every finding maps to a published journal requirement — not to the authors' preferences.

The tool is fully offline, never executes user code, and never modifies project files. It is designed to work in research computing environments that may be air-gapped or have restricted software installations.
