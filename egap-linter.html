EGAP Linter

AI-assisted replication package linter for social science

repro-lint is an open-source Python tool that audits replication packages before journal submission. It checks folder structure, data management, PII safety, and code quality for R and Stata — producing a concise, human-readable report that tells researchers exactly what to fix. It runs fully offline and never modifies your files.

Developed under an EGAP Innovation Grant · Open Science and Reproducibility · Co-PIs: Brayan A. Condori (GRADE) and Ronny M. Condor (Wyss Academy for Nature) · Research Assistant: Valeria Gonzales Torres

Quick start

# Install pip install replication-linter # Run on your project repro-lint /path/to/your/replication-package # Pre-submission check (strict mode) repro-lint . --strict

Requires Python 3.10+. Works on macOS, Linux, and Windows.

What it checks

Module 1 — Folder structure

Detects missing README files, absent standard directories, duplicate files, and orphan files. Verifies the README contains all seven sections required by the SSDE Template README v1.1 (endorsed by AEA, JPE, APSR, AJPS).

Module 2 — Data & PII safety

Opens CSV, Stata .dta, and Excel files and flags columns whose names or values suggest personally identifiable information. Supports Spanish and English identifiers (DNI, cédula, CPF, RUT, CURP, SSN, nombre, email, phone). Uses Stata variable labels — not just column names.

Module 3 — Code quality (R + Stata)

Static analysis only — never executes code. Flags absolute paths, missing random seeds, unpinned R packages, absent Stata version declarations, and dead file references across all .R, .Rmd, .do, and .ado files.

All checks

Severity levels: ERROR (blocks compliance), WARNING (causes first-round rejection), INFO (best-practice suggestion).

Configuration

Create a .linterrc file in TOML format at the project root to customise behaviour.

[linter] pii_sample_rows = 300 suppressed_codes = ["F008", "C014"] ignore_patterns = ["*.log", "*.tmp"] [linter.folder] expected_top_dirs = ["00_admin", "01_data", "02_code", "03_output"] [linter.pii] pii_deep_scan = false
